3) Especificação funcional

3.1 Objetivo

A página da proposta deve permitir montar uma proposta comercial com cálculo automático de mensalidade, serviços adicionais, treinamento, setup e desconto, exibindo um resumo final claro para validação e envio.

3.2 Regras de negócio

Plano e pessoas

O sistema deve aceitar apenas os planos:Basic
PRO
Ultimate
Desktop


O campo deve ser exibido como Quantidade de Pessoas
A quantidade deve ser sempre arredondada para a próxima faixa de 10exemplo: 11 → 20
exemplo: 51 → 60


O valor mensal deve ser obtido pela tabela correspondente ao plano e à faixa

Serviços adicionais

O usuário pode selecionar:Licença Facial
Gestão de Arquivos
Controle de Férias
Mais Requis Cálc Int


O valor deve ser calculado de acordo com:faixa de pessoas
plano escolhido
regra da tabela de preços


O sistema deve mostrar:valor de cada serviço
total consolidado dos serviços adicionais


O campo Taxa Única (R$) não deve aparecer

Treinamento e setup

A seção deve se chamar Escolha o tipo de treinamento
Ao selecionar o tipo, o valor correspondente deve ser preenchido automaticamente
O setup deve ser exibido em posição posterior ou anterior conforme o layout definido
O desconto deve existir em duas formas:valor em reais
percentual


Ao digitar um, o sistema deve calcular automaticamente o outro

Resumo da proposta

O resumo deve exibir:

Valor da Mensalidade
Valor do Setup de Implantação
Treinamento
Serviços Adicionais detalhados
Desconto
Total final

3.3 Critérios de aceite

A proposta é calculada corretamente para qualquer combinação válida
A faixa de pessoas é sempre arredondada para cima
O resumo mostra cada item individualmente
O desconto é sincronizado entre R$ e %
Os nomes dos campos seguem o padrão definido
